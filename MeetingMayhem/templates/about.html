Mission:

You are an agent who needs to reach agreement on a time and place to meet with your fellow agent, to exchange an essential asset. However, there is a malicious adversary trying to prevent you from reaching consensus on the time and place. Your only communication with your fellow agent is over a network, which is controlled by the adversary. At the end of the game session, you will vote to decide on the meeting time and place.

Goal:

Reach agreement with your fellow agent on the meeting time and place.

Cryptographic Tools:

  • Symmetric Encryption: Encrypt or decrypt your message using a Shared Symmetric Key known by you and the recipient.

    Consider that Alice wants to send a confidential message to Bob. One way would be to have a key that is only known to Alice and Bob and encrypt and decrypt the message with that key. We call that key the Shared Symmetric Key of Alice and Bob. Alice would encrypt the message with the key, and Bob would decrypt with that same key. When the same key is used in both encryption and decryption, we call the method symmetric encryption.     Read more
  • Asymmetric Encryption: Encrypt messages using the Receiver’s Public Key, or decrypt the message using the Receiver's Private Key.

    Consider that Alice wants to send a confidential message to Bob. Another method can be useful called asymmetric encryption. Each user locally generates a pair of keys (public key, private key) with a key-generation algorithm. Then, the public keys are shared with all users on the network, and each person keeps their private key to themself. For Alice to send a message to Bob, Alice encrypts the message with Bob’s public key, and Bob decrypts the message using his own private key.     Read more
  • Signature: Sign your message with the Sender’s (Your) Private Key; the receiver can verify your identity with the Sender’s (Your) Public Key. Here, a valid signature also encrypts and guarantees message integrity (i.e., that the message has not been modified in transit).

    Whereas encryption protects the confidentiality of the message, signatures help the receiver verify that the message actually came from the purported sender. For signatures in this game, we use the same public-private key pairs generated for asymmetric encryption. Alice can sign a message with her private key, and Bob can verify the signature with Alice’s public key, verifying that the message actually came from Alice.     Read more
